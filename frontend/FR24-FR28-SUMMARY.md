## What Was Skipped and Why

### FR-26: OAuth Import Trigger UI
- **Skipped**: Adding a "Sync Now" button to manually re-trigger GitHub/HuggingFace data import
- **Why**: The backend already triggers import automatically when a link is created or updated (via OAuth callback or manual URL save). A manual re-trigger button would require a new backend endpoint that doesn't exist yet. The current flow (connect → auto-import) matches the SRS requirement.

### FR-27: AI Tag Accept/Reject Workflow
- **Skipped**: Adding explicit "Accept" / "Reject" buttons for AI-suggested tags
- **Why**: The backend already persists AI-generated tags to the profile automatically during analysis. The user can remove unwanted AI tags using the existing "Remove" button in edit mode. The SRS says "auto-tagging is the default behavior" — tags are auto-applied, not suggested for approval.

### FR-28: Profile-Wide Public/Private Toggle
- **Skipped**: A single toggle to make the entire profile private
- **Why**: The SRS explicitly states "Developers may not set their entire profile to private" and "All profiles are public by default." The section-level visibility toggles (name, age, experience, education, projects) fulfill the SRS requirement for "toggle individual profile sections between public."

### Backend: Profile Visibility Filtering for Non-Owners
- **Skipped**: Updating the `_build_profile_read()` function to filter hidden entries for non-owner views
- **Why**: The `GET /profile/me` endpoint is owner-only. The recommendation system in `team.py` already has its own visibility filtering (lines 714-746). There is no public profile view endpoint in the current codebase, so filtering in `_build_profile_read` would hide entries from the owner who needs to manage them.

## Manual Testing Guide

### FR-24: Profile Fields
1. Navigate to `/profile`
2. Click "Edit profile"
3. Verify all fields are editable: name, age, years of experience
4. Verify education entries show: institution, degree, major, graduation year
5. Verify project history shows: name, description, role, duration, technologies
6. Verify external links section shows GitHub and HuggingFace connections
7. Verify skill tags section shows all tags
8. Click "Save" on name/age/experience — verify changes persist after page reload

### FR-25: Project History Entries
1. Navigate to `/profile` → Edit mode
2. Click "Add project" → fill in all fields (name, role, duration, technologies, description)
3. Click "Create" → verify project card appears
4. Click "Edit" on a project → modify fields → click "Save" → verify changes
5. Click "Delete" on a project → verify it's removed
6. Verify technologies display as individual tags on the card

### FR-26: Auto-Import from GitHub/HuggingFace
1. Navigate to `/profile` → Edit mode → External links
2. Enter a GitHub URL (e.g., `https://github.com/octocat`) → click "Save URL"
3. Wait a few seconds → verify "GitHub Data" panel appears with:
   - Commit count (12 months)
   - Public repositories count
   - Top repositories list with stars, forks, language
4. Verify parse status badge shows "Success" (green)
5. Repeat for HuggingFace URL → verify model count and dataset count appear
6. Test OAuth flow: click "Connect on GitHub" → authorize → verify redirect back to profile with data populated

### FR-27: AI Auto-Tagging
1. Navigate to `/profile` → verify existing skill tags
2. Look for tags with a purple "AI" badge — these were generated by the AI analysis
3. Click "Edit profile" → verify AI tags show the purple badge in edit mode
4. Click "Remove" on an AI tag → verify it's removed
5. Click "Re-analyze Profile" button → wait for analysis to complete
6. Verify new AI-suggested tags appear with the purple badge
7. Add a manual skill tag (type in search box, press Enter or click "Add typed skill")
8. Verify the manual tag does NOT have the "AI" badge

### FR-28: Section Visibility Toggles
1. Navigate to `/profile` → Click "Edit profile"
2. **Name field**: Click the eye icon next to "Name" → verify name shows "Not set" in view mode; click again to show
3. **Age field**: Click the eye icon next to "Age" → verify age is hidden; click again to show
4. **Experience field**: Click the eye icon next to "Years of experience" → verify it's hidden
5. **Education**: Click the eye icon on an education card → verify the card becomes semi-transparent (50% opacity); exit edit mode → verify the entry is still visible (owner always sees all entries)
6. **Projects**: Click the eye icon on a project card → verify the card becomes semi-transparent; exit edit mode → verify visibility
7. Verify hidden state persists after page reload
